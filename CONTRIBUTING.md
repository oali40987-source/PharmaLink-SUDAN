# Contributing to PharmaLink-SUDAN

Thank you for considering contributing to the PharmaLink-SUDAN project! We welcome contributions of all kinds, including bug reports, feature requests, and code contributions.

## How to Contribute

### Reporting Issues
If you find a bug or have a feature request, please open an issue in the GitHub repository. Include as much detail as possible to help us understand the problem.

### Making Changes
1. **Fork the repository**: Open the repository page on GitHub and click on the 'Fork' button.
2. **Create a new branch**: Create a new branch for your changes using `git checkout -b feature-branch-name`.
3. **Make your changes**: Implement your changes, ensuring that you follow the project's coding conventions.
4. **Write tests if applicable**: If you add new features or fix bugs, include tests to confirm your changes.
5. **Commit your changes**: Use `git commit -m "Descriptive commit message"` to commit your changes.
6. **Push your changes**: Push your changes to your forked repository using `git push origin feature-branch-name`.
7. **Create a pull request**: Go to the original repository and create a pull request from your feature branch, describing your changes and why you think they should be merged.

## Guidelines
- Please ensure your code adheres to the project’s style guidelines.
- Write descriptive commit messages to help us understand your changes.
- Be respectful and courteous in your interactions with others.

Thank you for contributing to PharmaLink-SUDAN! Your efforts help improve the project and support the community.
